# 时间投资账本 - Cursor开发需求文档

> **产品定位**：帮助用户量化时间价值，分析时间ROI，提升个人效率的投资工具
> **平台**：macOS（主力）+ iOS（辅助）
> **技术栈**：SwiftUI + Swift + Core Data + CloudKit/iCloud
> **收费模式**：订阅制（¥98/年或¥12/月）

---

## 一、产品核心价值

### 1.1 为什么做这个产品？

**用户痛点**：
- 不知道时间的真实价值
- 无法量化时间投入的回报
- 缺乏投资视角的时间管理

**产品解决方案**：
- 用时薪量化时间价值
- 用ROI分析时间投资回报
- 用投资顾问视角提供正向激励

### 1.2 核心差异化

| 传统时间管理 | 时间投资账本 |
|-------------|-------------|
| 只记录时间 | 计算时间价值 |
| 完成率评估 | ROI回报率分析 |
| 无亏损概念 | "刷短视频=亏损¥XX" |
| 批评者语气 | 投资顾问语气 |

## 二、功能需求

### 2.1 核心功能模块

#### 模块1：自动追踪系统

**macOS端（主力）**：
- 自动追踪应用使用时长
- 自动追踪文档编辑（VS Code、Pages等）
- 跨浏览器网站追踪（Safari/Chrome/Firefox/Edge）
- 自动分类和标记

**iOS端（辅助）**：
- 手动记录非Mac时间
- 查看数据概览
- 快捷添加备注

#### 模块2：价值计算系统

**核心公式**：
```
时间价值 = 时薪 × 时长 × 效率系数
ROI = (实际价值 - 预期价值) / 预期价值 × 100%
```

**计算逻辑**：
- 用户设置基础时薪
- 不同分类可设置不同效率系数（学习=1.2，娱乐=0.5）
- 自动生成每日/每周时间价值

#### 模块3：ROI分析系统

**分析维度**：
- 分类时间分布（工作/学习/娱乐占比）
- 时间价值趋势（周对比、月对比）
- 高价值时间占比（ROI>100%的时间）
- 投资建议（基于历史数据的建议）

**报告类型**：
- 日报：今日时间价值、高价值时间占比
- 周报：本周投资回报、分类分布、趋势分析
- 月报：月度ROI、习惯追踪、长期趋势

#### 模块4：轻量备注系统

**备注场景**：
- 时间记录附加简短备注
- 标签系统（#项目名、#客户名）
- 不做完整笔记系统，保持轻量

#### 模块5：数据导出系统

**导出格式**：
- Markdown：便于粘贴到Obsidian/Notion
- CSV：便于数据分析
- JSON：便于技术用户处理

**导出内容**：
- 选择时间范围（日/周/月/自定义）
- 包含时间记录+价值计算+备注
- 格式化美观，可直接使用

#### 模块6：订阅支付系统

**免费版**：
- 基础时间追踪
- 本地数据存储
- 基础统计图表

**Pro版（¥98/年或¥12/月）**：
- 跨浏览器网站追踪
- iCloud多设备同步
- ROI投资分析
- 数据导出
- 周报/月报生成
- AI洞察建议（后期）

```
│              View Layer                  │

└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│           ViewModel Layer                │
│        (@Observable 类)                 │

└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│           Service Layer                  │
│        (核心业务逻辑)                    │
│  TrackingService, ValueCalculator等     │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│          Repository Layer                │
│        (数据访问抽象)                    │
│   TimeRecordRepository, SettingsRepo    │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│            Data Layer                    │
│   Core Data + CloudKit + Keychain       │
└─────────────────────────────────────────┘
```

### 3.2 核心模块划分

| 模块 | 职责 | 关键技术 |
|------|------|----------|
| **Tracking** | 自动追踪应用/网站/文档 | Accessibility API, AppleScript |
| **Value Calculator** | 时间价值计算 | 纯Swift算法 |
| **Sync** | iCloud多设备同步 | Core Data + CloudKit |
| **Export** | 数据导出 | Markdown/CSV/JSON生成 |
| **Subscription** | 订阅管理 | StoreKit 2 |

### 3.3 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| **SwiftUI** | Apple原生，macOS/iOS共享代码，现代化UI开发 |
| **Core Data** | 本地数据持久化，与CloudKit无缝集成 |
| **CloudKit** | iCloud同步，无需自建服务器，Apple生态原生支持 |
| **StoreKit 2** | Apple原生订阅支付，无需第三方支付 |
| **Accessibility API** | 追踪前台应用，读取窗口标题 |

## 四、数据模型设计

### 4.1 核心实体

#### TimeRecord（时间记录）
```
- id: 唯一标识
- startTime: 开始时间
- endTime: 结束时间
- duration: 持续时长
- category: 分类（工作/学习/娱乐等）
- appName: 应用名称（自动追踪时）
- websiteUrl: 网站URL（浏览器追踪时）
- documentName: 文档名称（文档追踪时）
- hourlyRate: 时薪快照
- timeValue: 时间价值（计算得出）
- efficiencyScore: 效率系数
- note: 用户备注
- tags: 标签数组
- source: 来源（手动/自动追踪）
```

#### UserSettings（用户设置）
```
- hourlyRate: 基础时薪
- defaultEfficiencyFactor: 默认效率系数
- autoTrackingEnabled: 是否启用自动追踪
- trackApps: 是否追踪应用
- trackBrowsers: 是否追踪浏览器
- trackDocuments: 是否追踪文档
- syncEnabled: 是否启用同步
- subscriptionTier: 订阅等级
```

#### Category（分类）
```
- id: 唯一标识
- name: 分类名称
- color: 颜色
- icon: 图标
- defaultEfficiency: 默认效率系数
- countTowardsValue: 是否计入时间价值
```

### 4.2 实体关系

```
UserSettings (1) ──────> (N) TimeRecord
                              │
                              │ (N:1)
                              ▼
Category (1) ───────────> (N) TimeRecord
```

## 五、核心技术难点

### 5.1 跨浏览器追踪

**问题**：如何追踪Safari/Chrome/Firefox/Edge的网站访问？

**解决方案**：
1. **Accessibility API**：监听窗口变化
2. **AppleScript**：获取浏览器当前标签页URL
3. **浏览器历史数据库**：读取访问历史（Safari/Chrome支持）

**实现思路**：
```
1. 检测前台应用是否为浏览器
2. 如果是浏览器，执行AppleScript获取当前标签页URL
3. 定期采样（每5秒），记录用户在哪个网站
4. 按时间聚合，生成时间记录
```

**浏览器识别**：
- Safari: `com.apple.Safari`
- Chrome: `com.google.Chrome`
- Firefox: `org.mozilla.firefox`
- Edge: `com.microsoft.edgemac`

### 5.2 iCloud同步

**问题**：如何实现macOS和iOS数据实时同步？

**解决方案**：
- 使用Core Data + CloudKit集成
- 配置NSPersistentCloudKitContainer
- 处理同步冲突

**关键点**：
- 数据模型需要支持CloudKit
- 离线时本地存储，联网后自动同步
- 冲突处理：最后修改时间优先

### 5.3 macOS菜单栏常驻

**问题**：如何在菜单栏实时显示时间价值？

**解决方案**：
- 使用NSMenuExtra或NSStatusItem
- 后台追踪服务
- 定期更新菜单栏显示

**显示内容**：
- 今日时间价值："¥800"
- 今日时长："6h 30m"
- 或两者组合："¥800 | 6h"

## 九、注意事项

### 9.1 隐私合规

- Accessibility权限需要明确告知用户用途
- 浏览器历史属于敏感数据，需用户明确授权
- 数据存储在iCloud，遵循Apple隐私政策

### 9.2 性能优化

- 追踪采样间隔5秒，避免频繁读取
- 数据库查询优化，使用索引
- 大量数据时分页加载

### 9.3 用户体验

- 首次使用引导用户设置时薪
- 权限请求时解释用途
- 正向激励，避免制造焦虑
