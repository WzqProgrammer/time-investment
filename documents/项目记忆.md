# 项目记忆（time-investment）

> 用途：沉淀项目长期上下文，保证每次功能修改前后都有一致的认知与记录。

## 1. 项目目标
- 项目名称：`time-investment`
- 当前阶段：初始化阶段（SwiftUI/Xcode 项目骨架已创建）
- 核心目标：构建“时间投资账本”应用，支持时间投入记录、分析与可视化。

## 2. 关键目录与文件
- `time-investment/`：应用主代码目录（SwiftUI）
- `time-investmentTests/`：单元测试目录
- `time-investmentUITests/`：UI 测试目录
- `documents/时间投资账本_Cursor开发需求.md`：需求主文档
- `.gitignore`：仓库忽略规则

## 3. 当前已落地能力（持续维护）
- Xcode 工程与 SwiftUI 默认入口已建立
- `.gitignore` 已补充常见 Xcode/macOS 忽略项
- 已建立 Phase 1 的基础分层骨架：`Models / Services / Repositories / Features`
- 已实现 P0 可运行闭环：手动新增时间记录、价值计算、今日概览（时长/价值/ROI）、基础设置页
- 已补充 macOS 自动追踪基础服务骨架（Accessibility 权限检查 + 前台应用采样）
- 记录仓库默认切换为 Core Data（编程模型方式），支持本地持久化与变更监听
- 新增 macOS 菜单栏状态项，实时显示今日价值与时长，可快捷启停自动追踪
- 自动追踪支持“前台应用切换聚合为时间记录”的基础闭环
- 已补充 `ValueCalculator` 单元测试（时间价值、ROI、日报汇总）
- `UserSettings` 已接入 `UserDefaults` 持久化（启动读取、保存回写）
- 设置页支持分类效率系数自定义（按分类调整并参与价值计算）
- 已补 iOS 基础 Tab 骨架（今日/记录/统计/设置，共享核心逻辑）
- 已补仓库层基础单测（Core Data in-memory 的 CRUD 与变更通知）
- 自动追踪新增浏览器 URL 采样骨架（Safari/Chrome 的 AppleScript URL 读取）
- 统计页已接入最近 7 天维度数据（总价值、总时长、ROI、分类占比）
- 已实现 CSV 导出 MVP（最近 7 天，导出到临时目录并返回路径）
- 已实现首次引导（首启时薪设置 + 权限说明 + 自动追踪开关）
- 补充周统计单测（`weeklySummary`）
- 浏览器 URL 采样已补到 Firefox/Edge（当前 Firefox 以窗口标题近似，后续再强化）
- 新增报告服务 `ReportService`（周报/月报 Markdown 生成）
- 导出能力扩展为 CSV + JSON + Markdown
- 已加入免费/Pro 能力开关点（浏览器追踪、高级导出、报告预览）
- 已补导出层单测（JSON/Markdown 生成）
- Firefox URL 采样增强：先尝试直接读取 active tab URL，失败再退回窗口标题
- 报告新增“投资建议”文案（基于 ROI、分类结构、有效记录时长）
- 导出新增“自定义时间范围”能力（CSV/JSON/Markdown）
- 订阅能力抽象新增 `SubscriptionService`（本地实现 + StoreKit 预留实现）
- Pro 门禁改为经 `AppContainer.canUse` 统一判断
- 补充报告建议单测（`reportContainsAdvice`）
- 已拆分独立“订阅中心”Tab（订阅等级与报告语气设置从设置页迁出）
- 报告建议支持语气模板（投资顾问/教练/中性）
- 新增月度趋势数据模型（最近30天逐日价值与时长）
- 导出新增字段包含/排除配置（应用名、网站URL、备注、来源）
- 补充导出字段开关测试与语气模板测试
- 设置页完成排版重构：改为卡片式分组、滑杆化效率配置、主操作按钮突出，提升可读性与可操作性
- 设置页新增响应式布局：宽屏双列（参数/追踪 与 效率并排），窄屏单列，提升 macOS 场景下的信息密度与可读性
- 新增报告仓储能力（`ReportRepository`），报告不再只临时导出，可在应用内留存与删除
- 新增“报告中心”Tab：支持生成周报/月报、查看历史报告、删除报告
- 记录页新增手动计时兜底流程（开始计时/停止保存），用于自动追踪受权限限制时的替代录入
- 补充月度趋势天数测试与报告仓储 CRUD 测试
- 报告中心支持详情查看、复制 Markdown、单条报告导出
- 记录页新增追踪状态面板（运行中对象、权限失败提示）
- 补充单条报告导出测试（`exportSingleReportToTemp`）
- 报告中心新增筛选与搜索（全部/周报/月报 + 标题/内容搜索）
- 导出后支持系统打开文件（macOS Finder定位）
- 追踪状态增强：显示持续时长（秒级刷新）与最近错误列表
- 追踪服务新增错误回调（权限失败、AppleScript 执行异常）
- 修复 Core Data 初始化崩溃风险：持久化加载失败时自动降级到 in-memory，避免 `fatalError` 导致 `EXC_BREAKPOINT`
- 新增存储降级可视化提示：当 Core Data 降级到 in-memory 时在设置页顶部显示告警
- 修复 Debug 启动崩溃路径：移除仓储初始化断言中断，并将主目标 Debug 的 `ENABLE_APP_SANDBOX` 调整为 `NO`
